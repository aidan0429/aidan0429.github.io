How Universal Design is implemented

How Screen Readers Interpret Websites

Screen readers are programs which read the underlying code/structure of a website and convert it into text which is read aloud to the user. They do not interpret the visual layout of a webpage like someone with sight would.

  • They identify headings (H1, H2, H3) to identify the structure and navigate the page.
  • They interpret semantic HTML and CSS, not the visual layout.
  • They use landmarks such as header, nav, and main.
  • They describe meaningful alt text for images and links.
<

Semantic HTML Structure

  • Proper headings: Only one H1. Others follow hierarchy.
  • Landmarks: <main>, <header>, <nav>, <footer>
  • Meaningful alt text: describes the content purposefully.

Specific Barriers to Avoid

  • Improper or low color contrast

    • This makes it hard for users with visual impairments to read the content.

  • Font size too small or poorly spaced

    • This also makes it hard for users with visual impairments to read the content.

  • Overlapping elements or messy layout

    • This can confuse users and make navigation difficult.

  • Not tested with accessibility checkers

    • This can lead to unknown accessibility issues.

  • Images missing alt text

    • This makes it impossible for users with screenreaders to understand images.

  • Unlabeled buttons or links

    • This makes it difficult for users with screenreaders for navigate the page.

Accessibility Testing Tools

  • SilktideAutomated accessibility scanner
  • WAVE Tool Finds contrast issues, missing alt text, structure
  • Keyboard testing Navigating with only TAB and ENTER

Mobile Content First

When designing for mobile first, we ensure that the content is accessible for all users.

  • No overlapping or zooming issues
  • Clean and readable hierarchy
  • Text remains readable on small devices

Keyboard Accessibility

  • Skip to content button at the top
  • All links reachable using the TAB key
  • No mouse required to use the webpage
  • Logical focus order

Conclusion

Universal Design requires thoughtful planning and testing. From semantic HTML to keyboard functionality and mobile-first layout, designing accessibly creates a better experience for everyone.
